# 网络错误解决方案

## 问题描述
点击"生成报销Word"后出现"网络错误，请检查后端服务是否启动"的提示。

## 解决步骤

### 步骤1：启动后端服务

**方法一：使用快速启动脚本（推荐）**
```bash
python quick_start.py
```
这个脚本会自动检查依赖、启动服务器，并验证是否成功。

**方法二：手动启动**
1. 打开命令行终端
2. 切换到项目目录
3. 运行：
   ```bash
   python server.py
   ```
4. 看到 "Uvicorn running on http://0.0.0.0:8000" 表示启动成功

### 步骤2：验证服务是否运行

在浏览器中访问：http://localhost:8000/
- 如果看到FastAPI的欢迎页面，说明服务正常运行
- 如果无法访问，说明服务没有启动成功

### 步骤3：检查常见问题

#### 问题1：端口被占用
如果看到端口被占用的错误：
```bash
# 查看占用8000端口的进程
netstat -ano | findstr :8000

# 结束进程（替换PID为实际的进程ID）
taskkill /PID <PID> /F
```

#### 问题2：依赖未安装
确保所有依赖都已安装：
```bash
pip install -r requirements.txt
```

#### 问题3：API密钥问题
检查 server.py 文件中是否包含正确的API密钥：
```python
ZHIPU_API_KEY = "e1726b4c8ff9472ca63193add519d253.9F5ghmSnO2OiEpau"
```

#### 问题4：防火墙阻止
Windows防火墙可能阻止了连接。尝试：
1. 关闭防火墙
2. 或在防火墙中允许Python通过

### 步骤4：测试连接

使用浏览器控制台（F12）查看详细错误：
1. 打开工具页面
2. 按F12打开开发者工具
3. 点击"生成报销Word"
4. 查看Console标签页的错误信息

### 步骤5：使用调试脚本

运行测试脚本来诊断问题：
```bash
python test_server.py
```

这个脚本会自动检查：
- 依赖是否安装
- 服务器是否启动
- 端口是否可用

## 常见错误信息及解决

### 错误：Connection refused
- **原因**：后端服务未启动
- **解决**：运行 `python server.py` 启动服务

### 错误：Failed to establish a new connection
- **原因**：无法连接到localhost:8000
- **解决**：检查服务器是否正在运行

### 错误：CORS policy
- **原因**：跨域请求被阻止
- **解决**：后端已配置CORS，此问题很少出现

### 错误：API调用失败
- **原因**：智谱API无法访问
- **解决**：检查网络连接，确保能访问open.bigmodel.cn

## 完整测试流程

1. 运行 `python quick_start.py`
2. 等待看到 "✓ 服务器启动成功!"
3. 用浏览器打开 `ai-reimburse-tool.html`
4. 填写一些基本信息
5. 上传一张图片文件
6. 点击"生成报销Word"
7. 应该能看到处理进度和结果

如果仍有问题，请查看浏览器控制台的详细错误信息。
