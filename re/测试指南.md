# AI报销工具测试指南

## 功能完成情况

### ✅ 已完成的功能

1. **智谱GLM-4V OCR API集成**
   - 已集成真实的OCR识别功能
   - 支持发票、订单、支付记录三种类型
   - 每种类型使用不同的提示词优化识别效果

2. **三路文件上传界面**
   - 蓝色区域：发票上传
   - 绿色区域：订单上传  
   - 紫色区域：支付记录上传
   - 每个区域支持拖拽和点击上传

3. **多类型文件处理**
   - 后端API支持分别处理三种文件
   - 前端界面区分显示不同类型文件
   - 文件预览带有类型标识

4. **Word文档分组显示**
   - 按文件类型分组显示票据信息
   - 每张票据单独显示其详细信息
   - 包含汇总信息和小计

### 🔄 测试步骤

#### 1. 启动服务

```bash
# 安装依赖（如果尚未安装）
pip install -r requirements.txt

# 启动后端服务
python server.py
```

看到 `Uvicorn running on http://0.0.0.0:8000` 表示服务启动成功。

#### 2. 打开工具

用浏览器打开 `ai-reimburse-tool.html` 文件。

#### 3. 测试OCR识别

准备三种类型的图片：
- 发票图片（包含金额、商户信息）
- 订单截图（包含商品、数量、价格）
- 支付记录（包含支付金额和时间）

**测试流程：**

1. 填写基本信息（报销事由、方式、个人信息等）
2. 分别上传三种类型的图片
3. 点击"生成报销Word"按钮
4. 观察处理状态和OCR识别结果
5. 查看生成的Word文档

#### 4. 预期结果

- **OCR识别**：应该能准确提取图片中的关键信息
- **数据展示**：在页面上显示识别出的结构化数据
- **Word文档**：生成的文档应该包含：
  - 基本信息部分
  - 分组的票据明细（发票、订单、支付记录）
  - 签领表部分

### 🐛 常见问题

1. **图片识别失败**
   - 检查图片是否清晰
   - 确保图片中的文字可辨认
   - 尝试重新上传

2. **网络错误**
   - 确保后端服务正在运行
   - 检查防火墙是否阻止了连接

3. **API调用失败**
   - API密钥已配置在 server.py 中
   - 检查网络是否能访问智谱AI服务

### 🔧 技术要点

1. **OCR API使用**
   - GLM-4V多模态模型
   - 根据文件类型调整提示词
   - 错误处理和降级方案

2. **文件处理**
   - 临时文件创建和清理
   - Base64编码传输
   - 多文件类型支持

3. **Word生成**
   - 分节显示不同类型票据
   - 表格和文本混合布局
   - 格式化输出

## 实际使用建议

1. **图片质量**：使用高分辨率、清晰的图片
2. **文件命名**：使用有意义的文件名，便于识别
3. **批量处理**：支持一次上传多张同类型文件
4. **实时反馈**：处理过程有清晰的进度提示
