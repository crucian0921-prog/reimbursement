

AI REIMBURSE ASSISTANT（智能报销材料生成器）PRD


1. 项目概述

1.1 产品定位

AI 驱动的校内报销材料生成工具（Web 端） 通过 “信息填写 + 发票上传 + AI
识别 + 规则校验 + Word 导出”，将传统人工整理报销材料流程自动化。

1.2 产品目标

构建一个极简网页工具，实现： 用户填写报销信息 + 上传发票 / 订单截图 → AI
自动识别内容 → 自动校验缺失项 → 生成符合校内财务规范的 Word 报销材料

1.3 设计原则

本产品采用： 纯白留白基底 + 模块化圆角轻卡片 + 低饱和单色文字层级 +
原生极简表单控件

UI 设计极度克制，以 “填报效率与信息准确性” 为唯一目标。

视觉策略：无装饰性元素，无拟物化，无复杂交互动效，所有视觉元素服务于
“报销填报行为”

强对齐财务 / 行政系统的稳定与可信感

适配场景： 校内行政 / 财务报销 / 学生组织报销系统


2. 目标用户与使用场景

2.1 用户画像

高校社团负责人 学生会 / 研究生会成员 经常处理活动报销的人
不熟悉财务规范格式的人

2.2 使用场景

场景：社团会议报销 传统方式：查模板 → 手动整理 Word → 检查漏项 →
多轮修改 本产品方式：填表 + 上传发票 → 一键生成 Word + 自动校验


3. 产品信息架构

单页面结构（核心） 页面：AI 报销材料生成器 页面分区： 🟦 ①
基础信息区（Form Card） 圆角卡片（primary input） 字段：
报销事由（input） 报销方式（select） 报销人姓名（input）
报销人学号（input） 联系方式（input） 活动时间（input）

🟦 ② 材料上传区（Upload Card） 极简上传组件： 发票图片上传（multiple）
订单截图上传 支付记录上传 支持拖拽上传

🟦 ③ AI 处理状态区（System Card） 显示： OCR 识别中 AI 结构化中
校验规则执行中 输出实时状态（无动画，仅文字状态变化）

🟦 ④ 校验结果区（Validation Card） 输出结构： ✔ 完整项 ❌ 缺失项 ⚠ 风险项
示例： ✔ 已识别发票 ✔ 金额匹配 ❌ 缺少审批截图 ⚠ 金额与明细存在偏差

🟦 ⑤ Word 导出区（Action Card） 唯一主按钮：【生成报销 Word】 次级： 下载
Word 查看结构化 JSON（开发模式）


4. 核心功能设计（MVP）

4.1 信息录入模块

结构化表单输入： 报销基础信息 活动说明 人员信息 输出：JSON 结构

4.2 OCR 识别模块

输入： 发票图片 支付截图 输出： 金额 商户名称 日期 商品明细（可选）

4.3 AI 结构化模块

合并： 用户表单数据 OCR 结果 输出统一结构：

json

{

"activity": "",

"user": "",

"amount": "",

"items": [],

"purpose": "",

"invoice_status": true}

4.4 合规性校验模块

规则逻辑 + AI 辅助判断： 检查项： 是否存在发票 金额是否完整
是否缺少用途说明 是否缺少明细 是否符合校内财务抬头信息

输出： ✔ / ❌ / ⚠ 三态结果

4.5 Word 生成模块

自动生成标准报销文档： 1. 封面： 报销事由:xxx 报销方式:xxx 给
xxx，报销总金额为 xxx 元 报销人姓名:xxx 报销人学号:xxx 联系方式:xxx
报销金额:xxx 元

2. 证明材料 报销人姓名:xxx 报销人学号:xxx 报销金额:xxx 元 明细：

表格

  ------ ----------- ----------- ----------- ------------ ------
  物品   数量 / 个   单价 / 元   优惠 / 元   总价（元）   备注
                                                          
                                                          
  合计                                       元           
  ------ ----------- ----------- ----------- ------------ ------

合计：xxx 元 事由：xxx 用途：xxx 签名： 日期：xxx 年 xx 月 x 日

3. 发票 1： 购买详情、支付记录 1 发票 2： 购买详情、支付记录 2

垫付事由 xxx 报销垫付事由 本人 xxx（学号：xxx）在 xxx 中垫付共计 xxx
元，具体用于以下事项： 1. 购买 xxxx 套，花费 xxx 元； 2. 购买 xxx
套，花费 xxx 元。 签名（垫付人本人）： 日期：xx 年 xx 月 x 日

签领表 活动策划


5. Web UI 设计规范（重点重写）

5.1 视觉风格

背景：#FFFFFF 卡片背景：#FAFAFA 边框：#EAEAEA 主文字：#111111
次级文字：#666666 提示文字：#999999

5.2 组件规范

卡片：圆角 12px，无弱阴影，padding24px 输入框：极简边框，focus
仅变色无动效 按钮：单色无渐变，hover 轻微透明

5.3 布局

单列居中垂直排布：Header → 基础信息 → 上传 → AI 状态 → 校验结果 →
导出按钮

5.4 UI 核心原则

所有视觉设计服务于信息填报效率与财务严谨性表达，完全去装饰化，强化结构感与可信度。


6. 技术方案（优化版）

6.1 推荐技术栈

前端：Next.js（App Router）+ TailwindCSS 后端：FastAPI（Python） AI
能力：智谱 GLM-4.5-air（多模态 OCR + 票据结构化解析）
文件生成：python-docx

6.2 系统架构

前端表单文件上传 → FastAPI 后端 → 智谱 GLM 多模态模型 → 规则校验引擎 →
Word 生成器 → 文件下载流


7. 核心约束（MVP）

不含登录、数据库、多用户权限、复杂动画


8. 验收标准

1.  支持多张票据拖拽上传

2.  智谱 GLM 识别发票金额、商户、明细清晰

3.  完整 / 缺失 / 风险三态提示正常展示

4.  导出 Word 排版严格匹配校内报销模板

5.  本地前后端完整可运行
