# AI Reimburse Assistant 技术兼 UI 设计规范文档

## 1. 页面布局与 UI 组件规范（UI Spec）

本产品采用**单列居中自适应布局**，整体视觉遵循“去装饰化、高可信度”的严谨财务风格
。

### 1.1 全局视觉系统 (Theme)

*

**Canvas 背景色:** `#FFFFFF`（纯白）

*

**卡片背景色:** `#FAFAFA`（极浅灰，用于区隔模块）

*

**边框线 (Border):** `1px solid #EAEAEA`

*

**圆角 (Border Radius):** 全局统一 `12px`

*

**内边距 (Padding):** 卡片内部统一 `24px`

* **文字层级 (Typography):**

* 主标题/重点字：`#111111` (Bold)

* 次级正文/表单标签：`#666666` (Regular)

* 占位符/提示文字：`#999999` (Regular)

### 1.2 核心组件交互行为

① 基础信息卡片 (Form Card)

* **排列方式:** 双列或单列网格布局，标签（Label）置顶，输入框居下。

* **原生表单控件:**

* 输入框 (Input)：高度 `40px`，边框 `#EAEAEA`，Focus 状态下边框变为
`#111111`，无任何位移动画 。

* 下拉选择 (Select)：右侧配原生灰色倒三角，选中态逻辑同 Input 。

② 材料上传卡片 (Upload Card)

*

**拖拽区域 (Dropzone):** * 虚线边框 `1px dashed #999999`，背景色
`#FAFAFA` 。

*
内部放置极简图标（+）及提示语：“支持拖拽或点击上传发票图片、订单及支付截图”
。

* **文件列表 (File List):**

* 上传后在下方以纯文本列表呈现文件名，右侧配 `[删除]`
文本按钮（颜色：`#666666`），无缩略图预览以保持绝对克制 。

③ AI 处理状态卡片 (System Card)

*

**逻辑控制:** 默认隐藏。仅在用户点击“生成报销Word”后静态展现在表单下方
。

*

**视觉呈现:** 纯文字状态机切换（非动画 Progress Bar） 。

*

`OCR识别中...` $\rightarrow$ `AI结构化中...` $\rightarrow$
`校验规则执行中...`

④ 校验结果卡片 (Validation Card)

* **状态三色文本（无图标背景，仅靠符号和文字颜色）:**

*

`✔ 完整项`：文字颜色 `#2E7D32` (深绿)

*

`❌ 缺失项`：文字颜色 `#C62828` (深红)

*

`⚠ 风险项`：文字颜色 `#EF6C00` (深橙)

*

**单行间距:** `16px`，左对齐，确保财务审查一目了然 。

⑤ 操作导出卡片 (Action Card)

*

**主按钮【生成报销Word】:** 宽度 100%，背景色 `#111111`，文字
`#FFFFFF`，无渐变 。Hover 状态下微弱透明度变为 `0.9` 。

*

**次级操作区:** *
【下载Word】按钮（默认禁用，合规校验通过/文件生成后激活，样式为纯白底黑边框）
。

* 【查看结构化JSON】文本链接（仅在开发模式或底部折叠层展示） 。
